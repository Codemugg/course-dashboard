PYTHON COURSE FOR BEGINNERS.

INTRODUCTION TO PYTHON PROGRAMMING:


SYNTAX:


Syntax is the set of rules, principles, and processes that govern the structure of sentences in a given language, specifically word order. Each programming language has a particular syntax, so does python.
Python is also liked by developers because of it’s clean syntax. The language avoids using unnecessary characters to indicate some specificity.

COMMENTS:


Comments are the statements written in English which describes what a particular code does. A comment is preceded by a symbol telling the programming language you are using, to ignore that line (or part of a line) of code. In Python, the pound ‘#’ symbol is used to create comments.
Programmers use comments to make a line of code easier to understand for whoever reads it. You can write anything in your comments as long as it’s one line long.

Note: You should use comments sparingly. Do not comment explaining every single line of code in your program.

Following is an example of a good comment:

|
| #Calculates area of Circle
| def circle():
|   Radius = 5
|   Area = 3.14*pow(2, Radius)
| return Area
|
| Results = circle()
| print(Results)
Here, ‘ # Calculates area of circle ’ is a useful comment which tells us the purpose of defining that function i.e. to calculate and return the area of the circle.

CONSTANTS & VARIABLES:


Constants are the values which remain unchanged until the entire program is executed and terminated. Each number that we use in our program is a constant; i.e. the number 10 will always represent the value 10.
On the other hand, Variables are used to store and manipulate data to implement a flow or some specific logic.

| 100 * 100
| 20 + 20
| 10000
| 40
In some programming languages, you have to declare a variable by specifying the data type of the value that is to be stored. For an instance, in the C programming you create a variable like this: int a;
Creating and declaring a variable in python is lucid and straightforward.
Just pick a variable_name and assign a value to it using the ‘ = ’ symbol (assignment operator). Remember to try out these following examples in the Interactive shell.

|
| var = 150
| print(var)
| name = 'Ben'
| print(name)
| 150
| Ben
Here, the assignment operator i.e. ‘ = ’ sign, assigns the right value to the left variable_name. So, make sure that you write all the values towards the right side of the assignment operator and variable names on the left.

You can name variables whatever you'd like, as long as you follow these rules:
1. Variables can't have spaces. If you want to use two words in a variable, put an underscore between them:
i.e.,my_variable = "A string!" .
2. Variable names can only contain letters, numbers, and the underscore symbol.
3. You cannot start a variable name with a number. Although, you can start a variable with an underscore.
4. You cannot use Python keywords for variable names.

DATA TYPES:


Every variable in python has a specific type depending upon the value of data stored in the variable. In Python, each data value, like 2 or "Hello, World!", is called an object. You’ll learn about the objects later in this course, for now think about objects as a data value which has a name, data type and attributes.
Python has dynamic typing, which means you don't have to explicitly declare the type of your variable -- but if you want to, you can.
Firstly, Let’s try to understand how we can check the data type of a particular variable.
For this purpose, python comes with an inbuilt function called type(), which takes a variable name as an argument and determines its data type.
Take a look over following sample programs:

| num = 110
| print(type(num))
| string = 'Welcome To Codemugg'
| print(type(string))
| < class 'int' >
| < class 'str' >
In the above example, the ‘print(type(num))’ determines the type of variable num and displays it on the shell . The output ‘ < class 'int' > ’ denotes that the variable holds an int value i.e. an integer value.

Now, as you have learnt how to check the type of a particular variable. Lets explore the data types in Python.

Numeric Data Type:

There are three types of numeric types: int, float, and complex.

Int denotes any integer or numeric value that can either be positive or negative. Whole numbers (1, 2, 3, 4, etc.) and all the negative numbers(-1, -2, -3,etc) have the data type int, which is short for integer.

| num = 110
| print(type(num))
| < class 'int' >

Complex data types are used to represent a complex number.

| complex_num = 25+4j
| print(type(complex_num))
| < class 'complex' >


Float represents a decimal value( that is a number with a decimal point). 2.1, 8.2, and 9.9999 are all objects with the float data type. They are called floating point numbers. Like all data types, floating-point numbers have unique properties and behave in a certain way, similarly to integers.

| float_num = 93.66
| print(type(float_num))
| < class 'float' >


Boolean Values:

Boolean are the objects with bool data type, and have a value of True or False.

| bool_var = 93.66
| print(type(bool_var))
| < class 'bool' >


None Data Type :

Objects with a data type NoneType always have the value None. They are used to represent the absence of value.

| None


String :

String is the data type which represents a collection of various characters. There are many string manipulation techniques, which you’ll learn in the third section of this course. A string in python always begins and ends with either a single quotation or a double quotation.

| my_string = 'I love Coding.'
| print(my_string)
| print(type(my_string))
| I love Coding.
| < class 'str' >


Lists :

Lists in python can represent an ordered collection of different elements. Lists are mutable which means they can be modified after their creation.

| my_list = ['Code', 12, 25.3]
| print(my_list)
| print(type(my_list))
| ['Code', 12, 25.3]
| < class 'list' >


Tuples :

Tuples are just like lists which represent an ordered collection of elements. Unlike lists, Tuples are immutable which means they can’t be modified once they are created.

| my_tuple = ('Code', 12, 25.3)
| print(my_tuple)
| print(type(my_tuple))
| ('Code', 12, 25.3)
| < class 'tuple' >


Sets :

Sets are a collection of non-repetitive unordered elements which aren’t indexed.

| my_set = {'Tim','Bill','Steve'}
| print(my_set)
| print(type(my_set))
| {'Tim', 'Bill', 'Steve'}
| < class 'set' >


Dictionaries :

Dictionaries don't guarantee the order of the elements and are mutable. One important characteristic in dictionaries is that you can set your own access keys for each element.

| my_dict = {Name: 'Tim', Contact: '***969699856'}
| print(my_dict)
| print(type(my_dict))
| {Name: 'Tim', Contact: '***969699856'}
| < class 'dict' >


TYPE CASTING :


Type casting refers to conversion of one data type into another. It can be done using an explicit conversion method.

This way you can turn an int into a str, or a float into int.
For an instance,
Lets convert an integer value into a string using str().

| int_number = 55
| print(type(int_number))
| string_num = str(int_number)
| print(type(string_num))
| < class 'int' >
| < class 'str' >


The above code converts the integer value into a string Similarly, you can try converting a variable into integer using int() function, or you can convert it into float using float() function.

Implicit and Explicit conversions :

The example below shows implicit conversion while adding an int and a float.

| int_number = 45
| float_num = 45.20
| sum_intfloat = int_number + float_num
| print(sum_intfloat)
| print(type(sum_intfloat))
| print(int(sum_intfloat))
| 90.2
| < class 'float' >
| 90
Notice that sum_intfloat is float. Python uses float to avoid data loss since the int type can not represent the decimal digits.
If you still want your answer in integer format, you can use the int() function to convert it explicitly.
Remenber in some cases, you can’t use implicit conversion.
For example:
Take a look at following code, where we try to add an integer value and a string value.

| int_number = 15
| string = 32
| my_sum = int_number + string
| print(my_sum)
| Traceback (most recent call last):
| File "pyshell#0", line 1, in module
|   my_sum = int_number + string
| TypeError: unsupported operand type(s) for +: 'int' and 'str'
The above code gives an error because a string can’t be added to an integer. Hence, it becomes necessary to explicitly convert the string into integer before adding it to the number.

| int_number = 15
| string = 32
| my_sum = int_number + int(string)
| print(my_sum)
| 47
We don’t get any errors as we explicitly convert the string into integer before adding it to the number.

Precedence of Operators:
The order of operations is a set of rules used in mathematical calculations to evaluate an expression.
Remember, Please Excuse My Dear Aunt Sally? It is an acronym to help you remember the order of operations in math equations: parentheses, exponents, multiplication, division, addition, and subtraction. The parenthese overlaps the exponent operator, which overlaps multiplication and division, which overlaps addition and subtraction. If operators of equal precedence are used then the associativity rule is used which evaluates the expression from left to right.

For Example:

| 16 / 4 * 8
| 32.0
Here, the / and * operators have equal precedence. Hence, the expression is evaluated as 16 divided by 4 multiplied by 8. Thus, we get a result as 32.0.

Now let's evaluate the same operation using parentheses

| 16 / ( 4 * 8 )
| 0.5
Using the parentheses, The expression inside the parentheses that is (4 * 8) gets evaluated first and then the results get divided by 16.

Logical operators are another category of operators in Python. Like comparison operators, logical operators also evaluate to True or False.
