PYTHON COURSE FOR BEGINNERS.

FUNCTIONS IN PYTHON:



FUNCTIONS:


Functions are the best way to organize your code.

Using a function you can divide a complex program into small blocks of code, which makes it simpler to solve. The reusability of code can be done without repeatedly writing the code.

DECLARING & DEFINING A FUNCTION:


Python uses keyword ‘def’ to declare a function in a program.
Python doesn’t need an explicit declaration of a function as in ‘C’ Language .
To define a function, you just need to indent the block of code that is to be executed when the function is invoked.
Following is the syntax for declaring and defining a function in python:

| def FunctionName( PARAMETERS ):
|
|   BLOCK OF STATEMENTS
|
| FunctionName( ARGUMENTS )
You can implement the following program to understand how the function is defined, declared and invoked in python.

| def display():
|
|   print('You are inside the function.')
|
| print("You are going to enter in the function.")
| display()
| print("Program executed successfully..!")
| You are going to enter in the function.
| You are inside the function.
| Program executed successfully..!
|


ARGUMENTS:


You can define parameters between the parentheses next to the function name. When calling a function with parameters you have to pass arguments according to the parameters defined.
A function can either have a single argument or multiple arguments.
Single Argument:
To specify one parameter, just define it inside the parentheses next to the function name. In this example, the function my_number expects one number as an argument defined by the parameter num. The value of the argument is then accessible inside the function to be used.

| def my_number(num):
|
|   print(num)
|
| first_num = 150
| my_number(first_num)
| 110
Multiple Argument:
To specify multiple parameters, just define it inside the parentheses next to the function name. In this example, the function my_number expects two numbers as an argument defined by the parameter num1, num2. The value of the argument is then accessible inside the function.

| def my_number(first_num, second_num):
|
|   print(first_num, second_num)
|
| number1 = 150
| number2 = 200
| my_number(number1, number2)
| 110 200
This syntax of defining parameters and passing arguments is the same for any number of parameters. Note: The arguments have to be passed in the same order that the parameters are defined.
Default value :
You can set a default value for a parameter if no argument is given using the ‘=’ operator and a value of choice.
In this function, if no argument is given, the number 120 is assumed as the expected value by default

| def my_number(first_num=15):
|
|   print(first_num)
|
| my_number()
| 15
Lets try passing an argument in the above function.
| def my_number(first_num=15):
|
|   print(first_num)
|
| my_number(20)
| 20
The function then sets the value of first_num to 20 and then prints it.
Keyword or Named Arguments :

When calling a function, the order of the arguments have to match the order of the parameters.
The alternative is if you use keywords or named arguments.
Set the arguments to their respective parameters directly using the name of the parameters and the ‘=’ operators.
This example flips the arguments, but the function works as expected because you tell it which value goes to which parameter by name.

| def my_number(first_num,second_num):
|
|   print('The numbers are: ',first_num,second_num)
|
| my_number(second_num=150,first_num=200)
| The numbers are : 200 150
Any number of arguments: *args

If you don't want to specify the number of parameters, just use the * before the parameter name. Then the function will take as many arguments as necessary. The parameter name could be anything like *numbers, but there is a convention in Python to use *args for this definition of a variable number of arguments.

| def my_number(*args):
|
|   for i in args:
|     print(i)
|
| my_number(100,10,2)
| 100
| 10
| 2
Any number of Keyword/Named arguments: **kwargs

**kwargs Similar to *args, we can use **kwargs to pass as many keyword arguments as we want, as long as we use **.
Again, the name could be anything like **numbers, but **kwargs is a convention.

| def my_number(**kwargs):
|
|   for key,valur in kwargs.items():
|     print(key)
|     print(value)
|
| my_number(first_num=30,second_num=50)
| first_num
| 30
| second_num
| 50
You can also pass on different data types as arguments to other functions. Lets pass a string as an argument.
| def my_function(name):
|
|   print("My name is "+name)
|
| my_function('Sarah')
| My name is Sarah
Great..Now you know how to pass an argument in any function. Make sure you practice above programs.

Return :

A return keyword is used to return a value from a function.
In this example the function display_function() returns the string 'My Display function!'.
Notice that print() is a built-in function and our function is called from inside it.
The string returned by display_function() is passed as an argument to the print() function.

| def display():
|
|   return("My Display Function.")
|
| print(display())
| My Display Function.
Return multiple values:

Functions can also return multiple values at once. return_numbers() returns two values simultaneously.

| def display():
|
|   return(10,20)
|
| print(display())
| (10, 20)
Following program illustrates the addition of two numbers.

| def addition(x, y):
|
|   return(x+y)
|
| num1 = 50
| num2 = 50
| result = addition(num1, num2)
| print(result)
| 100

SCOPE OF VARIABLE :


The place where a variable is created defines its availability to be accessed and manipulated by the rest of the code. This is known as scope.
There are two types of scope: local and global.
Global Scope:

A global scope allows you to use the variable anywhere in your program. If your variable is outside a function, it has global scope by default.
| number = 10
| def my_num():
|
|   print('The number is',number)
|
| my_num()
| The number is 10.

Local Scope:

When you declare a variable inside a function, it only exists inside that function and can't be accessed from the outside.

| def my_num():
|
|   number = 10
|   print('The number is',number)
|
| my_num()
| The number is 10.
In the above code, If you try to access the number outside the function it will throw an error.

Mixing Scopes :

If you use the same name for variables inside and outside a function, the function will use the one inside its scope.
So when you call my_number(), the number=10 is used to print the phrase.
On the other hand, when calling print() outside the function scope, number=20 is used because of its global scope.

| number = 20
| def my_num():
|
|   number = 10
|   print('The number is',number)
|
| my_num()
| print('The number is',number)
| The number is 10.
| The number is 20.


BUILT-IN FUNCTIONS :


Python consists of many built-in functions. Built-In functions are functions which are pre-defined by python and are known by the python interpreter. You just have to use those functions, which makes it easier for you to develop a program.
In this section you will learn the most basic built-in functions in python.

Let's start with the input() function:

Input :


If you need to interact with a user when running your program in the command line (for example, to ask for a piece of information), you can use the input() function.
| name = input('Enter your name: ')
| print('Welcome to Codemugg !',name)
|
| Enter your name: Sarah | Welcome to Codemugg ! Sarah
Anything that is entered by the user, is treated as a string by input() function. Hence, if you want an integer input from the user, you will have to convert the input explicitly.

| num = int(input('Enter your number: '))
| print('You entered ',num)
|
| Enter your name: 50
| You entered 50
In a similar way, if you want a float value, you can convert it using explicit conversion.

USER DEFINED FUNCTIONS :


User Defined functions are the functions which are declared and defined by the user. All the functions that you have implemented in your programs are called as user defined functions.
For example,

| def display():
|   print("I'm a user defined function.")
|
| display()
| I'm a user defined function.

LAMBDA FUNCTION:


A Python lambda function can only have one expression and can't have multiple lines. It is supposed to make it easier to create some small logic in one line instead of a whole function as is usually done.
Lambda functions are also anonymous, which means there is no need to name them

The basic syntax is very simple: just use the lambda keyword, define the parameters needed, and use the ":" operator to separate the parameters from the expression.
The general forms is:
|
| lambda ARGUMENTs : EXPRESSION
|
Single parameter example :


Look at this example using only one parameter.

| cubic = lambda x: x**3
| print(cubic(5))
|
| 125
Multiple parameter :


If you want, you can also have multiple parameters.
Following program calculates the simple interest on the principal amount.

| simple_interest = lambda price*years*rate: (price*years*rate)//100
| print(simple_interest(10000,2,0.5))
|
| 100.00
Lets execute the same program by calling the lambda function directly, without using any variable.

| print((lambda price*years*rate: (price*years*rate)//100)(10000,2,0.5))
|
| 100.00

Using lambda functions with other built-in functions
Map :


The Map function applies the expression to each item in a list.
Let's calculate the cubic of each number in the list.

| numbers = [2, 5, 10]
| cubic = list(map(numbers, lambda x: x**3))
| print(cubic)
| [8, 25, 1000]
Filter :


The Filter function will filter the list based on the expression.
Let's filter to have only the numbers greater than 5.

| numbers = [2, 5, 7, 10]
| filtered_numbers = list(filter(numbers, lambda x: x < 10))
| print(filtered_numbers)
| [2, 5, 7]

EXCEPTION HANDLING:


Errors are a part of every developer’s life, and knowing how to handle those errors is a skill on it’s own.
The way Python deals with errors is called 'Exception Handling'.
If some piece of code runs into an error, the Python interpreter will raise an exception.

Types of Exceptions

Let's try to raise some exceptions on purpose and see the errors they produce.

TypeError

The best way to raise a TypeError is adding a string to integer value.
Lets try it.

| result = 'The string is '+15
| print(result)
| Traceback (most recent call last):
| File "pyshell#0", line 1, in module
|   'The string is'+15
| TypeError: can only concatenate str (not "int") to str
IndexError

Now, try to access an index that doesn't exist in a list.
A common mistake is to forget that sequences are 0-indexed, meaning the first item has index 0, not 1.
In this example, the list of subjects ends at index 2.
Lets try accessing the 4th element with index 3.


| subjects = ['Maths', 'English', 'Physics']
| print(subjects[3])
| Traceback (most recent call last):
| File "pyshell#2", line 1, in module
|   print(subjects[3])
| IndexError: list index out of range
NameError

If we try to print a variable that doesn't exist:


| print(variable_name)
| Traceback (most recent call last):
| File "pyshell#3", line 1, in module
|   print(variable_name)
| NameError: name 'variable_name' is not defined
ZeroDivision

Math doesn't allow division by zero, so trying to do so will raise an error, as expected.


| 7 / 0
| Traceback (most recent call last):
| File "pyshell#4", line 1, in module
|   7/0
| ZeroDivisionError: division by zero
Exception Handling

Now we know how to cause errors that will crash our code and show us some message saying something is wrong.
To handle these exceptions just make use of the try/except statement.


| try;
|   print(variable_name)
|
| except:
|   print('Variable not found.')
|
| Variable not found.
The example above shows the use of the try statement.

Put the block of code that may cause an exception inside the try scope.
If everything runs alright, the except block is not invoked.
But if an exception is raised, the block of code inside the except is executed.
This way the program doesn't crash and if you have some code after the exception, it will keep running if you want it to.
Specific Exception Handling

In the last example the except block was generic, meaning it was catching anything. Good practice it to specify the type of exception we are trying to catch, which helps a lot when debugging the code later.
If you know a block of code can throw an IndexError, specify it in the except:


| try;
|   subject = ['Maths', 'English', 'Physics']
|   print(subject[3])
|
| except IndexError:
|   print('Index out of range.')
|
| Index out of range
You can use a tuple to specify as many exception types as you want in a single except:

| try;
|   print('My Code !')
|
| except (IndexError, NameError, TypeError):
|   print('There is an error.')
|
| My Code !
You can also add an else command at the end of the try/except. It runs only if there are no exceptions.
| my_var = 'My Variable' | try;
|   print(my_var)
|
| except NameError:
|   print('NameError caught.')
| else: |   print('No NameError') |
| My Variable
| No NameError
Raising Exceptions

The raise command allows you to manually raise an exception.
This is particularly useful if you want to catch an exception and do something with it -- like logging the error in some personalized way like redirecting it to a log aggregator, or ending the execution of the code since the error should not allow the progress of the program.
If you know a block of code can throw an IndexError, specify it in the except:


| try;
|   raise IndexError('This index is not allowed.')
|
| except IndexError:
|   print('Doing something with exception')
|   raise
|
| Doing something with exception
| Traceback (most recent call last):
|   File "pyshell#5", line 2, in module
|     raise IndexError('This index is not allowed.')
| IndexError: This index is not allowed.
Finally

The finally block is executed independently of exceptions being raised or not.
They are usually there to allow the program to clean up resources like files, memory, network connections, etc.

| try;
|   print(my_var)
|
| except NameError:
|   print('Except Block')
| finally;
|   print('Finally Block')
|
| Doing something with exception
| Traceback (most recent call last):
| File "pyshell#5", line 2, in module
|   raise IndexError('This index is not allowed.')
| IndexError: This index is not allowed.
